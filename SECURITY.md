# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability in Achatina, please
report it privately.

Do not open a public GitHub issue for suspected security problems.

Instead, email:

- `seyiakadri@gmail.com`

Use a subject line such as:

- `Achatina security report`

Include:

- a description of the issue
- affected files, commands, or surfaces
- reproduction steps if available
- potential impact
- any suggested mitigation if you have one

## Disclosure Preference

Please allow time for the issue to be reviewed and fixed before any public
disclosure.

If the report is valid, coordinated disclosure is preferred.

## Scope

This policy is for security vulnerabilities in the public `achatina` code and
its published release surface.

General usage questions, licensing questions, and commercial inquiries should be
handled through the normal project contact path, not through the security
channel.

## Response Expectations

This project will try to acknowledge legitimate reports in a reasonable time,
but no formal SLA is currently promised.
